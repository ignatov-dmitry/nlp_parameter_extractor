# NLP Parameter Extractor API Guide

## Описание

API для автоматического извлечения параметров товаров из текста пользователя. Система анализирует естественный язык и возвращает структурированные данные о категории товара, его характеристиках и ценовом диапазоне.

## Установка и запуск

### Требования
- Python 3.8+
- FastAPI
- Uvicorn
- pandas
- fuzzywuzzy

### Установка зависимостей
```bash
pip install fastapi uvicorn pandas fuzzywuzzy python-Levenshtein openpyxl
```

### Запуск сервера
```bash
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Сервер будет доступен по адресу: `http://localhost:8000`

### Запуск через Docker

#### Требования
- Docker
- Docker Compose

#### Запуск контейнера
```bash
docker-compose up --build
```

Сервер будет доступен по адресу: `http://localhost:8000`

#### Остановка контейнера
```bash
docker-compose down
```

## Эндпоинты

### 1. POST /extract
Основной эндпоинт для извлечения параметров из текста.

#### Запрос
```bash
curl -X POST "http://localhost:8000/extract" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Нужен кожаный диван 2 метра на 2 секции"
  }'
```

#### Структура запроса
```json
{
  "text": "Нужен кожаный диван 2 метра на 2 секции"
}
```

#### Ответ (успешно)
```json
{
  "status": "success",
  "category_name": "мебель",
  "category_id": "11",
  "parameters": [
    {
      "name": "тип",
      "id": "27",
      "value": ["диван"],
      "value_id": "4007",
      "filter_name": "подкатегория"
    },
    {
      "name": "длина",
      "id": "31",
      "value": ["200"],
      "value_id": "4649",
      "filter_name": "длина, см"
    },
    {
      "name": "количество секций",
      "id": "27",
      "value": ["2"],
      "value_id": "191529",
      "filter_name": "подкатегория"
    }
  ],
  "price_min": null,
  "price_max": null,
  "debug_info": "КАТЕГОРИЯ: мебель (ID: 11)..."
}
```

#### Ответ (ошибка)
```json
{
  "status": "error",
```

## Структура JSON-ответа

### Основные поля
- **status** (string): Статус обработки (`success` или `error`)
- **category_name** (string): Название категории товара (например, "мебель", "освещение")
- **category_id** (string): ID категории (11 для мебели, 12 для освещения)
- **parameters** (array): Массив найденных параметров
- **price_min** (integer): Минимальная цена (если указана)
- **price_max** (integer): Максимальная цена (если указана)
- **debug_info** (string): Отладочная информация

### Структура параметра
```json
{
  "name": "тип",
  "id": "27",
  "value": ["диван"],
  "value_id": ["4007"],
  "filter_name": "подкатегория"
}
```

- **name** (string): Название параметра (как его назвал ИИ)
- **id** (string): ID параметра в базе данных
- **value** (array): Массив значений параметра
- **value_id** (array или string): ID значения(й) в базе данных. Может быть:
  - Массив строк при нескольких совпадениях: `["4007", "4009", "4013"]`
  - Одна строка при одном совпадении: `"4007"`
  - null если значение не найдено в базе
- **filter_name** (string): Официальное название фильтра в базе

## Логика поиска и обработки параметров

### Обработка числовых значений
- Для числовых параметров (количество плафонов, диаметр, высота) используется **строгое совпадение**
- Значение "2" соответствует только "2", а не "12" или "216"
- Проверка: `clean_value.isdigit()` - если true, используется точное совпадение

### Обработка текстовых значений
- Для текстовых параметров используется **fuzzy matching** с алгоритмом `fuzz.partial_ratio`
- Минимальный порог совпадения: **70%**
- Пример: "диван" находит "угловой диван", "прямой диван", "модульные диваны"

### Smart Threshold (умный порог)
- Если лучший результат имеет score >= 95% И разница между первым и вторым результатом > 10%, возвращается **одиночный результат**
- Если score < 95% ИЛИ разница < 10%, возвращаются **все найденные варианты** (до 5 штук)
- Это предотвращает спам при точных совпадениях и сохраняет выбор при нечетких запросах

### Фильтрация параметров
- В итоговый отчет добавляются только параметры, которые:
  1. Явно извлечены AI из текста пользователя, И
  2. Имеют найденные значения в базе данных
- Это предотвращает добавление нежелательных параметров (например, "материал: металл"), если они не были упомянуты пользователем

### Синонимы
- Система использует словарь синонимов (`data/synonyms.json`) для корректного сопоставления параметров
- Пример: "модерн" → "стиль"

## Особенности системы

### Мульти-выдача параметров
Система поддерживает возврат нескольких совпадений для одного параметра. Поле `value_id` может быть:
- **Массивом** при нескольких найденных вариантах: `["4007", "4009", "4013"]`
- **Строкой** при одном совпадении: `"4007"`
- **null** если значение не найдено в базе

Пример мульти-выдачи:
```json
{
  "name": "подкатегория",
  "id": "27",
  "value": ["диван"],
  "value_id": ["4007", "4009", "4013"],
  "filter_name": "подкатегория"
}
```

Это означает, что для запроса "диван" найдены три варианта:
- ID 4007: Прямые диваны
- ID 4009: Угловые диваны
- ID 4013: Модульные диваны

### Поддерживаемые категории
- **11 (Мебель)**: диваны, кровати, шкафы, столы, стулья, ковры, подушки и т.д.
- **12 (Освещение)**: люстры, бра, торшеры, светильники, фонари и т.д.

### Конвертация единиц измерения
- **Мебель**: все габариты конвертируются в сантиметры (2 метра → 200 см)
- **Освещение**: все габариты конвертируются в миллиметры (2 метра → 2000 мм)

### Поддержка синонимов
Система поддерживает 317+ синонимов для более точного распознавания параметров:
- "кожаный" → "кожа"
- "шерстяной" → "шерсть"
- "минималистичный" → "минимализм"

### Нечеткий поиск
При поиске параметров используется алгоритм нечеткого совпадения (fuzzy matching) с коэффициентом схожести > 85%.

## Интеграция в собственную систему

### Python
```python
import requests
import json

url = "http://localhost:8000/extract"
payload = {
    "text": "Нужен кожаный диван 2 метра на 2 секции"
}

response = requests.post(url, json=payload)
result = response.json()

print(f"Категория: {result['category_name']}")
print(f"Параметры: {result['parameters']}")
```

### JavaScript/Node.js
```javascript
const fetch = require('node-fetch');

const url = 'http://localhost:8000/extract';
const payload = {
  text: 'Нужен кожаный диван 2 метра на 2 секции'
};

fetch(url, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(payload)
})
.then(res => res.json())
.then(data => {
  console.log('Категория:', data.category_name);
  console.log('Параметры:', data.parameters);
});
```

### PHP
```php
<?php
$url = 'http://localhost:8000/extract';
$payload = json_encode([
    'text' => 'Нужен кожаный диван 2 метра на 2 секции'
]);

$ch = curl_init($url);
curl_setopt($ch, CURLOPT_CUSTOMREQUEST, 'POST');
curl_setopt($ch, CURLOPT_POSTFIELDS, $payload);
curl_setopt($ch, CURLOPT_HTTPHEADER, ['Content-Type: application/json']);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

$response = curl_exec($ch);
$result = json_decode($response, true);

echo "Категория: " . $result['category_name'];
echo "Параметры: " . json_encode($result['parameters']);
?>
```

4. Убедитесь, что база данных `data/db.xlsx` находится в правильном месте
